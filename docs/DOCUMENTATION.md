## 1. Constants

```c
#define IF   0   // Instruction Fetch stage index
#define ID   1   // Instruction Decode stage index
#define EX   2   // Execute stage index
#define MEM  3   // Memory Access stage index
#define WB   4   // Write Back stage index

#define COMMAND_LEN  5    // Max characters in an instruction mnemonic (null-terminator included)
#define COMMANDS_NUM 13   // Total number of supported instruction mnemonics
```

The stage index constants (`IF`–`WB`) are used to index `Pipeline.stage[]` and make hazard-detection conditions self-documenting.

## 2. Data Structures

### 2.1 `Control`

```c
typedef struct Control {
    // EX / ALU stage signals
    int RegDst;   // 1 = destination is rd (R-type), 0 = destination is rt (I-type)
    int ALUOp0;   // ALU operation control bit 0
    int ALUOp1;   // ALU operation control bit 1
    int ALUSrc;   // 1 = second ALU operand is sign-extended immediate, 0 = register

    // MEM stage signals
    int Branch;    // 1 = this instruction is a branch
    int MemRead;   // 1 = read a word from data memory
    int MemWrite;  // 1 = write a word to data memory

    // WB stage signals
    int RegWrite;  // 1 = write result to the register file
    int MemToReg;  // 1 = write-back value comes from memory, 0 = from ALU
} Control;
```

**Purpose:** Mirrors the control-signal set generated by the MIPS control unit for a single instruction. Each bit drives a specific multiplexer or enables a specific action in the corresponding pipeline stage.

---

### 2.2 `Command`

```c
typedef struct Command {
    char type;           // 'R', 'I', 'L', 'S', 'B', 'J', or '0' (bubble)
    char name[5];        // Null-terminated mnemonic, e.g. "add\0"
    int  address;        // Memory address of this instruction
    int  operand1;       // Destination register (binary-encoded decimal)
    int  operand2;       // First source register (binary-encoded decimal)
    int  operand3;       // Second source register or immediate value
    char command[30];    // Full instruction text as read from the trace file
    Control signals;     // Pre-set control signals for this instruction type
} Command;
```

**Type codes:**

| `type` | Meaning |
|--------|---------|
| `'R'`  | R-type arithmetic/logical (`add`, `sub`, `and`, `or`) |
| `'I'`  | I-type ALU with immediate (`addi`, `subi`, `ori`, `andi`) |
| `'L'`  | Load word (`lw`) |
| `'S'`  | Store word (`sw`) |
| `'B'`  | Branch (`beq`, `bneq`) |
| `'J'`  | Jump (`j`) — parsed but not fully simulated |
| `'0'`  | Bubble / stall — all control signals zero |

**Register encoding:** Registers are stored as *binary numbers encoded as decimal integers*. For example, register `$3` (binary `11`) is stored as the integer `11`. This encoding is produced by `convertToBinary()`.

---

### 2.3 `Pipeline`

```c
typedef struct Pipeline {
    Command stage[5];  // stage[0]=IF, stage[1]=ID, stage[2]=EX, stage[3]=MEM, stage[4]=WB
    int forward;       // 0 = no forwarding, 1 = forwarding enabled
    int branch;        // 0 = branch resolves in MEM (flush 2), 1 = branch resolves in ID (stall 1)
} Pipeline;
```

**Purpose:** Represents the entire pipeline state at a single point in time. Each element of `stage[]` holds the `Command` currently occupying that pipeline stage.

## 3. Global Variables

```c
char commands[][5] = {
    "add","sub","and","or",       // indices 0-3  → R-type
    "addi","subi","ori","andi",   // indices 4-7  → I-type ALU
    "lw",                         // index  8     → Load
    "sw",                         // index  9     → Store
    "beq","bneq",                 // indices 10-11 → Branch
    "j"                           // index  12    → Jump
};
```

Used by `getCommandType()` to map a mnemonic string to a type character.

```c
Control rControl    = { 1,1,0,0, 0,0,0,1,0 };  // R-type
Control lControl    = { 0,0,0,1, 0,1,0,1,1 };  // lw
Control sControl    = { 0,0,0,1, 0,0,1,0,0 };  // sw
Control bControl    = { 0,0,1,0, 1,0,0,0,0 };  // branch
Control iControl    = { 0,0,0,1, 0,0,0,1,0 };  // I-type ALU
Control stallControl= { 0,0,0,0, 0,0,0,0,0 };  // bubble/NOP
```

Pre-allocated `Control` values assigned to each instruction during parsing.

```c
float cycle  = 0;   // Total clock cycles elapsed (incremented each time printfCycleInfo is called)
float stalls = 0;   // Number of stall (bubble) instructions that have passed through WB
```

Used together to compute CPI at the end of simulation.

## 4. Function Reference

### 4.1 `main`

```c
void main(int argc, char* argv[])
```

**Purpose:** Program entry point. Initialises the pipeline, prompts the user to choose a trace file, sets forwarding/branch mode flags, starts the simulation, then prints CPI.

**Parameters:**
- `argc` — argument count (expected: 3, i.e., `./mips <forward> <branch>`).
- `argv[1]` — forwarding mode (`"0"` or `"1"`).
- `argv[2]` — branch resolution mode (`"0"` or `"1"`).

**Flow:**
1. Call `initPipe(&pipe)` to fill all 5 stages with empty `No Command` bubbles.
2. Prompt user with `scanf` and open the chosen trace file with `fopen`.
3. Assign `argv[1]` / `argv[2]` to `pipe.forward` / `pipe.branch`.  
   *(⚠ Bug: `argv[n]` is a `char*`; it should be `atoi(argv[n])`.)*
4. Call `nextCycle(traceFile, &pipe)` — the main simulation loop.
5. Call `printCPI()`.
6. Close the trace file.

---

### 4.2 `nextCycle`

```c
void nextCycle(FILE* traceFile, Pipeline* pipe)
```

**Purpose:** Main simulation loop. Each iteration of the `while(1)` loop represents one clock cycle.

**Parameters:**
- `traceFile` — open file handle pointing to the instruction trace.
- `pipe` — pointer to the current pipeline state.

**Algorithm (one iteration):**

```
1. Read the next instruction from the trace file.
2. If EOF marker received → call endFile() and return.
3. Shift pipeline: stage[4] ← stage[3] ← stage[2] ← stage[1] ← stage[0]
4. Place the new instruction in stage[IF].
5. Increment cycle counter and print pipeline state.
6. Hazard detection (see below).
```

**Hazard detection — No Forwarding (`pipe->forward == 0`):**

| Condition | Action |
|-----------|--------|
| `stage[EX].RegWrite == 1` AND `stage[ID]` reads a register that `stage[EX]` writes | Insert 1 stall after ID; print state |
| `stage[MEM].RegWrite == 1` AND `stage[ID]` reads a register that `stage[MEM]` writes | Insert 1 stall after ID; print state |
| Branch taken AND `pipe->branch == 0` (MEM resolution) | Call `flush1()` — flush 2 instructions |
| Branch taken AND `pipe->branch == 1` (ID resolution) | Insert 1 stall; print state |

**Hazard detection — With Forwarding (`pipe->forward == 1`):**

| Condition | Action |
|-----------|--------|
| `stage[EX].MemRead == 1` (load-use hazard) AND `stage[ID]` reads the loaded register | Insert 1 stall after ID; print state |
| Branch taken AND `pipe->branch == 0` | Call `flush1()` |
| Branch taken AND `pipe->branch == 1` | Insert 1 stall; print state |

**Branch-taken detection:** `pipe->stage[IF].address != pipe->stage[ID].address + 4`  
(The address of the instruction just fetched is not the sequential successor of the branch.)

---

### 4.3 `stall`

```c
void stall(Pipeline* pipe, int location)
```

**Purpose:** Insert a single pipeline bubble immediately after the stage at `location`, shifting all later stages forward by one position.

**Parameters:**
- `pipe` — pointer to the pipeline.
- `location` — index of the stage *before* which the bubble is inserted (typically `ID = 1`).

**Effect:** Stages `location+2` through `WB` shift one step toward WB. The slot at `location+1` is replaced with a bubble from `addBubble()`. The stage at `location` (and earlier) is **not** moved — it will be re-evaluated next cycle, effectively replaying it.

**Example:** `stall(pipe, ID)` with the pipeline `[IF, ID, EX, MEM, WB]` produces `[IF, ID, bubble, EX, MEM]`. `WB` is dropped.

---

### 4.4 `flush1`

```c
void flush1(Pipeline* pipe)
```

**Purpose:** Flush the 2 instructions that were speculatively fetched after a taken branch (when the branch resolves in the MEM stage). Inserts 2 bubble cycles.

**Effect:** Runs a loop twice. Each iteration shifts stages 2–4 forward by one position, sets `stage[1]` (the slot just above IF) to a bubble, increments the cycle counter, and prints the pipeline state. This drains the two incorrectly-fetched instructions out of the pipeline.

---

### 4.5 `flush`

```c
void flush(Pipeline* pipe)
```

**Purpose:** Flush 3 speculatively-fetched instructions (unused in the current simulation paths, but available for a branch that resolves even later). Inserts 3 bubble cycles using the same mechanism as `flush1` but iterates 3 times.

---

### 4.6 `bStall`

```c
void bStall(Pipeline* pipe)
```

**Purpose:** Insert 3 stall bubbles starting from the IF stage (index 0), shifting all 5 stages forward. Intended for a scenario where the entire pipeline must be stalled for 3 cycles. Currently not called by `nextCycle`.

---

### 4.7 `addBubble`

```c
Command addBubble()
```

**Purpose:** Construct and return a `Command` struct representing a pipeline bubble (NOP / stall).

**Returns:** A `Command` where:
- `type` = `'0'`
- `name` = `"stll"` *(truncated form of "stall" as written in the source)*
- `command` = `"stall"`
- `operand1/2/3` = `0`
- `signals` = `stallControl` (all signals zero)

Bubbles have all-zero control signals so they cause no reads, writes, or memory operations as they pass through the pipeline.

---

### 4.8 `getCommandType`

```c
char getCommandType(char* commandName)
```

**Purpose:** Identify the type of a MIPS instruction by matching its mnemonic against the global `commands[]` array.

**Parameters:**
- `commandName` — null-terminated mnemonic string read from the trace file.

**Returns:** One of `'R'`, `'I'`, `'L'`, `'S'`, `'B'`, `'J'` depending on which entry in `commands[]` matches.

**Algorithm:** Iterates over all 13 entries in `commands[]`. For each entry it compares characters one-by-one until a mismatch, end-of-string on both sides (match), or end of `commandName`. On a full match it uses the array index to determine the category:

| Index range | Returned type |
|-------------|---------------|
| 0 – 3       | `'R'`         |
| 4 – 7       | `'I'`         |
| 8            | `'L'`         |
| 9            | `'S'`         |
| 10 – 11     | `'B'`         |
| 12           | `'J'`         |

---

### 4.9 `convertToBinary` / `toBin`

```c
int  convertToBinary(int origin)
long toBin(int dno)
```

**Purpose:** Convert a non-negative decimal integer to its binary representation encoded as a decimal integer.

**Example:** `convertToBinary(3)` → `11` (binary `11` stored as decimal `11`).

**Algorithm:** Repeatedly takes `origin % 2` to extract the least-significant bit, multiplies it by a growing power of 10, and accumulates the result.

**Note:** `toBin` is a duplicate of `convertToBinary` with a `long` return type. Both implement the same algorithm. Neither is used for actual binary arithmetic — the result is a *display* representation.

**Limitation:** Only works correctly for small non-negative integers. Large register numbers or immediate values may overflow.

---

### 4.10 `printCPI`

```c
void printCPI()
```

**Purpose:** Print the Cycles Per Instruction ratio at the end of simulation.

**Formula:**

```
CPI = cycle / (cycle - stalls - 4)
```

- `cycle` — total clock cycles (every call to `printfCycleInfo` increments this).
- `stalls` — number of bubble instructions that reached the WB stage (counted inside `printfCycleInfo`).
- `4` — the four drain cycles added by `endFile` are subtracted so they do not distort the instruction count.

---

### 4.11 `printfCycleInfo`

```c
void printfCycleInfo(Pipeline pipe)
```

**Purpose:** Print the current state of all 5 pipeline stages for the current clock cycle, and count stalls.

**Output format:**

```
Clock N:
Fetch instruction: <command>
Decode instruction: <command>
Execute instruction: <command>
Memory instruction: <command>
Writeback instruction: <command>
```

**Side effect:** If `pipe.stage[WB].command` equals `"stall"`, the global `stalls` counter is incremented. This is the mechanism by which stall cycles are tallied for the CPI calculation.

---

### 4.12 `endFile`

```c
void endFile(Pipeline* pipe)
```

**Purpose:** Drain the pipeline after the last real instruction has been fetched. Inserts 4 bubble (NOP) cycles so all real instructions can complete their WB stage.

**Algorithm:** Loops 4 times. Each iteration:
1. Shifts stages 1–4 forward by one position.
2. Sets `stage[0]` to a bubble.
3. Sets the name and command string of the stage now at position `j` to `"Empty"` / `"No Command"`.
4. Increments the cycle counter and prints pipeline state.

*(⚠ Bug: variable `j` ends at `0` after the inner loop, so `stage[0]` is always overwritten, not the stage that was just shifted out.)*

---

### 4.13 `readCommand`

```c
Command readCommand(FILE* filePtr)
```

**Purpose:** Parse one line of the trace file and return a fully populated `Command` struct.

**Parameters:**
- `filePtr` — open file handle positioned at the start of a line.

**Returns:** The parsed `Command`, or the result of `endOfFile()` if `EOF` is reached.

**Parsing steps:**

1. Read the decimal address with `fscanf`.
2. Skip the space after the address with `fseek(filePtr, 1, SEEK_CUR)`.
3. Read characters into `name[]` until a `$` or space is encountered (the first `$` signals the start of operands).
4. Null-terminate `name` and call `getCommandType` to set `type` and the matching `Control` signals.
5. Read operands using `fscanf` for integers and `fseek` to skip separators. The exact sequence depends on the instruction type:
   - **R-type** (`operand1`=rd, `operand2`=rs, `operand3`=rt): skip 1 char before each; read 3 registers.
   - **I-type ALU** (`operand1`=rt, `operand2`=rs, `operand3`=imm): skip 2 chars then 1 char between operands.
   - **Load** (`operand1`=rt, `operand2`=imm, `operand3`=rs): skip 1, then 2 chars between.
   - **Store** (`operand2`=rt, `operand3`=imm, `operand1`=rs): store order is inverted vs. lw.
   - **Branch** (`operand1`=rs, `operand2`=rt, label skipped with `%s`).
6. Seek back to `init + 5` (5 bytes past the start of the line) to re-read the full command text.
7. Read characters into `command[]` until `\n` or `EOF`.
8. Return the completed `Command`. If `ch == EOF`, return `endOfFile()` instead.

---

### 4.14 `endOfFile`

```c
Command endOfFile()
```

**Purpose:** Return a sentinel `Command` value that signals the main loop in `nextCycle` to stop reading.

**Returns:** A bubble command (from `addBubble()`) with `name` set to `"END OF FILE"`.

**Detection:** `nextCycle` checks for this sentinel with `strcmp(newCommand.name, "END OF FILE") == 0`.

---

### 4.15 `initPipe`

```c
void initPipe(Pipeline* pipe)
```

**Purpose:** Initialise all 5 pipeline stage slots to a neutral "No Command" state before simulation begins.

**Algorithm:** Iterates over `stage[0]` through `stage[4]`. For each slot, calls `addBubble()` to fill it with a zero-control-signal bubble, then overwrites `name` with `"Empty"` and `command` with `"No Command"` so that the very first printed cycles show meaningful labels instead of `"stall"`.

## 5. Data-Flow Diagram

```
Trace File
    │
    ▼
readCommand()
    │  (one instruction per cycle)
    ▼
┌─────────────────────────────────────────┐
│               Pipeline                  │
│  IF ──► ID ──► EX ──► MEM ──► WB       │
│  [0]    [1]    [2]    [3]    [4]        │
└─────────────────────────────────────────┘
    │
    ├─ Hazard detected?
    │      ├─ Data hazard (no fwd) ──► stall()    → 1 bubble after ID
    │      ├─ Load-use hazard (fwd) ──► stall()   → 1 bubble after ID
    │      ├─ Taken branch (MEM) ──► flush1()     → 2 bubbles
    │      └─ Taken branch (ID)  ──► stall()      → 1 bubble after ID
    │
    └─ EOF? ──► endFile() → 4 drain bubbles → printCPI()
```

Each call to `printfCycleInfo` records a snapshot of all 5 stages and counts stalls reaching WB.

## 6. Known Issues & Limitations

| # | Location | Description |
|---|----------|-------------|
| 1 | `main` | `pipe.forward = argv[1]` assigns a `char*` to an `int`. Use `atoi(argv[1])`. Same for `pipe.branch`. |
| 2 | `main` | The `switch` on `option` opens `traceFile` inside case-scoped blocks; the variable goes out of scope immediately. The file that is actually used is always `trace1.txt` (hard-coded on line 103). |
| 3 | `endFile` | The inner `for` loop leaves `j = 0`, so `strcpy(&pipe->stage[j].name, "Empty")` always targets `stage[0]` instead of the stage that just left the pipeline. |
| 4 | `endOfFile` / `initPipe` | `strcpy(&end.name, "END OF FILE")` passes the *address of a pointer* rather than the buffer; should be `strcpy(end.name, "END OF FILE")`. Same pattern in `initPipe` and `endFile`. |
| 5 | `getCommandType` | Returns no value if no mnemonic matches (undefined behaviour in C). |
| 6 | `convertToBinary` | Register numbers > 9 produce decimal strings that are not valid binary (e.g., register 10 → `1010` fits in `int`, but register 16 → `10000` and larger values can overflow). |
| 7 | Jump (`'J'`) | Jump instructions are parsed (type returned) but no operand-reading code exists for them; the `switch` in `readCommand` has no `case 'J'`. |
| 8 | `flush` / `bStall` | These functions are defined but never called by `nextCycle`. They may be intended for future extension. |
